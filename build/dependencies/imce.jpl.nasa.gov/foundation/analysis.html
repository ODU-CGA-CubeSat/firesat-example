Abstract

The Analysis Ontology defines general concepts and properties for analyses (e.g., trade studies, driving requirements analysis, etc.). It provides a basis for specialization by domain experts.

3.4. MeasuredElement

A MeasuredElement is a model element that may be measured by one or more associated Metric s. Both Objective s and Requirement s are MeasuredElement s.

Super entities: AnalyzedElement, CharacterizedElement

Specializations: project:Concern, mission:Objective, mission:Requirement

Relations having MeasuredElement as range:

  • Criterion measures MeasuredElement

5. Concepts

5.1. Analysis

Analysis is a deprecated synonym for Explanation .

Super entities: Explanation

Specializations: CostEstimate, DrivingRequirementsExplanation, KeyRequirementsExplanation, TradeStudy

Supertype Relations having Analysis as domain:

  • Analysis analyzes AnalyzedElement

  • Analysis explains CharacterizedElement

  • Analysis validates CharacterizedElement

5.2. Assumption

An Assumption is Characterization that is taken to be true for the purpose of analysis.

Super entities: Characterization

Specializations: Fact

Relations having Assumption as domain:

  • Assumption limits CharacterizedElement

Supertype Relations having Assumption as domain:

  • Assumption characterizes CharacterizedElement

5.3. Characterization

A Characterization is an element that provides a parametric or other characterization of another model element. The canonical example of a Characterization is a set of name/value pairs, but other characterizations (e.g., classification) are possible.

Super entities: AnalyzedElement, CharacterizedElement, base:IdentifiedElement

Specializations: Assumption, project:DevelopmentState, fse:EngineeringChangeRequest, Explanation, fse:FunctionAnnotation, GlossaryEntry, fse:Implementer, fse:MassConstraint, Quantity

Relations having Characterization as domain:

  • Characterization characterizes CharacterizedElement

Supertype Properties: base:hasAlternateName, base:hasCanonicalName, base:hasDescription, base:hasIdentifier, base:hasIndexEntry, base:hasShortName, base:hasSortKey, base:hasUuid

5.4. Constraint

A Constraint is a container for one or more mathematical expressions asserting constraints on values.

Super entities: AnalyzedElement, CharacterizedElement

Relations having Constraint as domain:

  • Constraint constrains ConstrainedElement

5.5. CostEstimate

A CostEstimate is a kind of Explanation that provides a cost estimate for the model elements it explains.

Super entities: Analysis

5.6. Criterion

A Criterion establishes a region for some Metric that corresponds to success for some MeasuredElement . A Criterion for the Metric sample mass in kg might be at least 10 kg .

Super entities: AnalyzedElement, CharacterizedElement, base:IdentifiedElement

Relations having Criterion as domain:

  • Criterion measures MeasuredElement

Relations having Criterion as range:

  • Metric hasCriterion Criterion

Supertype Properties: base:hasAlternateName, base:hasCanonicalName, base:hasDescription, base:hasIdentifier, base:hasIndexEntry, base:hasShortName, base:hasSortKey, base:hasUuid

5.7. DrivingRequirementsAnalysis

DrivingRequirementsAnalysis is a deprecated synonym for DrivingRequirementsExplanation .

Super entities: DrivingRequirementsExplanation

5.8. DrivingRequirementsExplanation

A DrivingRequirementsAnalysis provides the rationale for designating a set of Requirement s as driving , and makes the designation via the explains property.

Super entities: Analysis

Specializations: DrivingRequirementsAnalysis

  • Restricts range of explains to be an instance of Requirement

5.9. Explanation

An Explanation is a product that captures or summarizes the results of an analysis activity and relates it to one or more other model elements. It may contain narrative prose directly or provide a reference to external products. Driving requirements analyses, trade studies, and cost estimates are all examples of Explanation .

Super entities: base:AggregatedElement, Characterization

Specializations: Analysis, fse:GangConfiguration

Relations having Explanation as domain:

  • Explanation analyzes AnalyzedElement

  • Explanation explains CharacterizedElement

  • Explanation validates CharacterizedElement

Supertype Relations having Explanation as domain:

  • Explanation aggregates AggregatedElement

  • Explanation characterizes CharacterizedElement

5.10. Fact

A Fact is an Assumption that is true.

Super entities: Assumption

Supertype Relations having Fact as domain:

  • Fact limits CharacterizedElement

5.11. GlossaryEntry

GlossaryEntry is a Characterization that associates a term with a definition.

Super entities: Characterization

Supertype Relations having GlossaryEntry as domain:

  • GlossaryEntry characterizes CharacterizedElement

Direct Properties: hasGlossaryDefinition, hasGlossaryTerm

5.12. KeyRequirementsAnalysis

KeyRequirementsAnalysis is a deprecated synonym for KeyRequirementsExplanation .

Super entities: KeyRequirementsExplanation

5.13. KeyRequirementsExplanation

A KeyRequirementsAnalysis provides the rationale for designating a set of Requirement s as key , and makes the designation via the explains property.

Super entities: Analysis

Specializations: KeyRequirementsAnalysis

  • Restricts range of explains to be an instance of Requirement

5.14. Metric

A Metric establishes a method for quantifying achievement or satisfaction of one or more MeasuredElement s. Note that the relationship is many to many: a given Metric may jointly quantify multiple MeasuredElement s. Likewise, a MeasuredElement may be quantified by multiple Metric s. Appropriate Metric s for the Objective image Martian north polar region might include data volumes at specified wavelength and resolution.

Super entities: AnalyzedElement, CharacterizedElement, base:IdentifiedElement

Relations having Metric as domain:

  • Metric hasCriterion Criterion

Relations having Metric as range:

  • Quantity hasMetric Metric

Supertype Properties: base:hasAlternateName, base:hasCanonicalName, base:hasDescription, base:hasIdentifier, base:hasIndexEntry, base:hasShortName, base:hasSortKey, base:hasUuid

5.15. Quantity

A Quantity attaches a value for some Metric to one or more SpecifiedElement s. That is, it evaluates some aspect of a design by assigning a definite value to a specific Metric .

Super entities: Characterization

Relations having Quantity as domain:

  • Quantity hasMetric Metric

  • Quantity hasValue QuantityValue

  • Quantity quantifies SpecifiedElement

Supertype Relations having Quantity as domain:

  • Quantity characterizes CharacterizedElement

5.16. QuantityValue

A QuantityValue assigns a value to a Quantity .

Super entities: AnalyzedElement, CharacterizedElement

Relations having QuantityValue as range:

  • Quantity hasValue QuantityValue

5.17. TradeStudy

A TradeStudy explains some set of model elements and their properties through comparison with alternatives.

Super entities: Analysis

7. Relations

7.1. Analyzes

Attributes: Asymmetric, Irreflexive

Source: Explanation

Target: AnalyzedElement

Forward: analyzes An Explanation a analyzes some IdentifiedElement e if and only if a considers or otherwise takes account of e .

Reverse: isAnalyzedBy

Super terms: AnalyzedElement, CharacterizedElement

Sub terms: fse:HasGangMember

7.2. Characterizes

Attributes: Asymmetric, Irreflexive

Source: Characterization

Target: CharacterizedElement

Forward: characterizes A Characterization c characterizes a CharacterizedElement e if and only if c describes, delimits, or restricts e .

Reverse: isCharacterizedBy See characterizes .

Super terms: AnalyzedElement, CharacterizedElement, mission:SpecifiedElement

Sub terms: Explains, Limits

7.3. Constrains

Attributes: Asymmetric, Irreflexive

Source: Constraint

Target: ConstrainedElement

Forward: constrains A Constraint c constrains some ConstrainedElement e if and only if c imposes restrictions on one or more properties of e .

Super terms: AnalyzedElement, CharacterizedElement

7.4. Explains

Attributes: Asymmetric, Irreflexive

Source: Explanation

Target: CharacterizedElement

Forward: explains An Explanation a explains some SpecifiedElement e if and only if a provides a rationale or justification for some design aspect of e .

Reverse: isExplainedBy See explains .

Super terms: AnalyzedElement, CharacterizedElement, Characterizes

Sub terms: fse:HasGangAggregate, Validates

7.5. HasCriterion

Attributes: InverseFunctional

Source: Metric

Target: Criterion

Forward: hasCriterion

A Metric m hasCriterion some Criterion c if and only if c establishes a region of success for m .

Reverse: isCriterionFor

Super terms: AnalyzedElement, CharacterizedElement

7.6. HasMetric

Attributes: Functional, Asymmetric, Irreflexive

Source: Quantity

Target: Metric

Forward: hasMetric

An Quantification q hasMetric some Metric m if and only if q assigns a value to m for some aspect of a design.

Super terms: AnalyzedElement, CharacterizedElement

7.7. HasValue

Attributes: Functional, Asymmetric, Irreflexive

Source: Quantity

Target: QuantityValue

Forward: hasValue

Super terms: AnalyzedElement, CharacterizedElement

7.8. Limits

Attributes: Asymmetric, Irreflexive

Source: Assumption

Target: CharacterizedElement

Forward: limits

Reverse: isLimitedBy See limits .

Super terms: AnalyzedElement, CharacterizedElement, Characterizes

7.9. Measures

Attributes: Asymmetric, Irreflexive

Source: Criterion

Target: MeasuredElement

Forward: measures

A Criterion c measures some MeasuredElement e if and only if c defines a region for some Metric that corresponds to succes for e .

Reverse: isMeasuredBy

Super terms: AnalyzedElement, CharacterizedElement

7.10. Quantifies

Attributes: Asymmetric, Irreflexive

Source: Quantity

Target: mission:SpecifiedElement

Forward: quantifies

A Quantification q quantifies some SpecifiedElement e if and only if q assigns a value to some Metric for e .

Reverse: isQuantifiedBy

Super terms: AnalyzedElement, CharacterizedElement

7.11. Validates

Attributes: Asymmetric, Irreflexive

Source: Explanation

Target: CharacterizedElement

Forward: validates An Explanation a validates some SpecifiedElement e if and only if a provides a definitive, determinative explanation for some design aspect of e .

Reverse: isValidatedBy See validates .

Super terms: AnalyzedElement, CharacterizedElement, Explains
